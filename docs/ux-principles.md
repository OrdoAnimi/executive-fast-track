# UX Principles

## Product feel

Executive Fast Track should feel like an executive learning cockpit, not a generic course marketplace. The interface should be calm, premium, restrained, and boardroom-grade.

The design should support focus, confidence, and quick orientation. It should avoid childish gradients, noisy cards, excessive animation, motivational clutter, and generic SaaS visual language.

## Visual direction

Preferred direction:

- Charcoal, dark navy, off-white, muted gold, steel blue, and neutral grey.
- Strong typography with generous spacing.
- Clear hierarchy and high readability.
- Structured cards for modules and executive lenses.
- Minimal animation.
- Dashboard-like confidence without clutter.

## Interaction principles

The learner should always know:

- What module they are in.
- What level they are working through.
- Whether they are in low-energy, study, or performance mode.
- What artefact the module produces.
- What the next useful action is.

## Neurodivergent-friendly design

The course should reduce activation friction. It should support short sessions, skimming, return later behaviour, visible progress, and multiple input modes.

Avoid forcing long walls of text, mandatory quizzes, hidden navigation, ambiguous next steps, or excessive cognitive switching.

## Page model

Core pages:

- Home.
- Course.
- Learning Model.
- Executive Lenses.
- Copilot.
- Resources.
- Simulator.

## Content density

The interface should present concise executive summaries first, with expandable or linked deeper material. The learner should be able to operate from the top layer, then go deeper when capacity allows.

## Tone

Direct, practical, serious, executive. Avoid hype, motivational coaching language, and academic over-explanation.
